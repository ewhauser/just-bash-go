# Commands

gbash ships with 60+ built-in commands. Every command is backed by a **command registry** -- if a command is not registered, it cannot execute. Unknown commands never fall through to host binaries.

The built-in set targets GNU coreutils flag parity. See the [compatibility matrix](https://ewhauser.github.io/gbash/compat/latest/) for per-flag coverage.

## Default Registry

Call `DefaultRegistry()` to get the stock command set:

```go
registry := gbash.DefaultRegistry()

gb, err := gbash.New(gbash.WithRegistry(registry))
```

You can also register additional commands on top of the default set (see [Contrib Commands](/commands/contrib) and the [custom-zstd example](https://github.com/ewhauser/gbash/tree/main/examples/custom-zstd)).

### File and Path

| Command | Description |
|---|---|
| `basename` | Strip directory and suffix from a filename |
| `cat` | Concatenate and print files |
| `chmod` | Change file mode bits |
| `chown` | Change file owner and group |
| `cp` | Copy files and directories |
| `dirname` | Strip the last component from a path |
| `du` | Estimate file space usage |
| `file` | Determine file type |
| `find` | Search for files in a directory hierarchy |
| `ln` | Create symbolic links |
| `link` | Create a hard link |
| `ls` | List directory contents |
| `dir` | List directory contents (ls-style) |
| `mkdir` | Create directories |
| `mv` | Move or rename files |
| `readlink` | Print the target of a symbolic link |
| `rm` | Remove files or directories |
| `rmdir` | Remove empty directories |
| `stat` | Display file status |
| `touch` | Change file timestamps or create empty files |
| `tree` | List directory contents in a tree format |

See [File and Path](/commands/file-and-path) for details.
